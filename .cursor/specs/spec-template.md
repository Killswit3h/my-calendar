# Specification Template

Use this template when creating new `spec.md` files for `/write-spec` and related flows.

```
# Specification: [Feature Name]

## Goal
[1-2 sentences describing the core objective]

## User Stories
- As a [user type], I want to [action] so that [benefit]
- [Optional] Additional user story (max two total)

## Specific Requirements

**[Requirement name]**
- [Up to 8 concise bullet points describing the technical or architectural expectations for this requirement]

[Repeat for up to 10 requirements]

## Visual Design
[If there are mockups, list them here]

**`planning/visuals/[filename]`**
- [Up to 8 bullets describing UI expectations for this mockup]

[Repeat for each visual asset]

## Existing Code to Leverage

**Code/logic/component name**
- [Up to 5 bullets explaining what the existing code does and how to reuse it]

[Repeat up to 5 code areas]

## Out of Scope
- [Up to 10 concise statements describing what should not be done for this spec]
```
